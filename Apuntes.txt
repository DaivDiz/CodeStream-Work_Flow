El flujo moderno: Productividad

Tradicional vs Moderno
Tradicional
-Herramientas no integradas
-Revision del codigo en Poll Request (PR)
-Mas lineas de comando
-Cambio de contexto entre sitio web y editor
-Comunicacion externa al editor
-Documentación externa al editor
Moderno
-Herramientas integradas en el editor
-Revision de código en el PR y en cualquier líneas de código
-Revision de código antes del commit
-Mas automatizado
-Menos cambios de contexto
-Comunicacion integrada 
-Documentación integradas

Flujo rama (branch) tradicional
-Create a branch
-Open a pull request
-Merge and deploy

Flujo rama (branch) morderno
-Asign ticket
-Create branch
-Notify to team

->Escribir código
->Sugerencias (FeedBack Request)
->Revision de código pre-PR
->Aprobación Final

Pull Request o FeedBack (Shift Left)
Flujo Personal
-Pull Request en editor
-Mejor diff
-Comentarios adicionales
-Eficiencia administrativa
Flujo Equipo
-Metodología compartida
-Revision de código automatizada
-Comentarios adicionales
-Más informal, mas efeiciente
